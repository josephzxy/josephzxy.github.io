总括

  • 浅层神经网络
  • 和word2vec中的cbow非常类似，将待预测的中心词换成了label
  • 在简单文本分类问题上可以达到State-of-art的效果（非常接近或略逊于最优解）
  • 训练和测试时间比深度神经网络少了几个数量级
  • 适合对句子中词语顺序不敏感的分类任务

结构

fasttext的结构和下图十分类似，不过去掉了中间的Hidden linear layer

简单来说fasttext可以被分成三层

  • 输入层
  • 隐藏层
  • 输出层

为了方便说明，我们定义以下常量

VOCAB_SIZE表示词汇表大小。

EMBEDDING_DIM表示经过embedding层输出，每个词被分布式表示的向量的维度，这里设置为100。比如对于“达观”这个词，会被一个长度为100的类似于[ 0.97860014, 5.93589592, 0.22342691, -3.83102846, -0.23053935, …]的实值向量来表示；

MAX_WORDS表示一篇文档最多使用的词个数，因为文档可能长短不一（即词数不同），为了能feed到一个固定维度的神经网络，我们需要设置一个最大词数，对于词数少于这个阈值的文档，我们需要用“未知词”去填充。比如可以设置词汇表中索引为0的词为“未知词”，用0去填充少于阈值的部分；

CLASS_NUM表示分类问题的类别数。

输入层

输入层，即embedding层

输入的shape为[BATCH_SIZE, MAX_WORD]

输入的是用词汇表简单向量化后的一段文本，比如“我爱吃香蕉”被向量化为[10, 30, 50, 700, 1000]，向量中的每个数值是该单词在词汇表中的索引

输出的shape为[BATCH_SIZE, MAX_WORD, EMBEDDING_DIM]

即把输入数据中的每一个单词，从用一个词汇表索引表示，转换成了这个词对应的词向量，词向量维度为EMBEDDING_DIM

隐藏层

输入的shape是[BATCH_SIZE, MAX_WORD, EMBEDDING_DIM]

输出的shape是[BATCH_SIZE, EMBEDDING_DIM]

即把这个文本中每个单词的词向量进行了叠加平均，作为这个文本的向量。比如一句话中有两个单词，分别对应的词向量是[1, 1]和[5, 5]，那么这句话的向量就是[3, 3]

输出层

输出层，即SOFTMAX分类层

输入的shape是[BATCH_SIZE, EMBEDDING_DIM]

输出的shape是[BATCH_SIZE, CLASS_NUM]

即针对这个文本的向量，计算其落入每一个类别的概率。

FastText的一些Tricks

Hierarchical softmax

基于Huffman tree, 用于加速输出层softmax计算的速度

可加速训练过程，因为训练集已知label， 只需要计算输入数据被分类到该label的概率即可（不是很确定）

测试过程仍然要计算所有的值

bag-of-N-gram features

分为

  • word n-gram （单词级）
  • char n-gram （字符级）
